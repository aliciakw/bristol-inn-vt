## Trust Boundaries

| Boundary                      | Description                                                                                             |
| ----------------------------- | ------------------------------------------------------------------------------------------------------- |
| PRISMIC_TOKEN → API request   | Token flows from astro:env/server into createClient options; never emitted to client                    |
| Prismic slug param → getByUID | slug comes from URL param in [slug].astro; passed to getByUID which only queries Prismic — no SSRF risk |

## STRIDE Threat Register

| Threat ID | Category               | Component                           | Disposition | Mitigation Plan                                                                                                                           |
| --------- | ---------------------- | ----------------------------------- | ----------- | ----------------------------------------------------------------------------------------------------------------------------------------- |
| T-02-08   | Information Disclosure | PRISMIC_TOKEN in logs               | mitigate    | No console.log of token; getClient() only passes it to createClient options                                                               |
| T-02-09   | Tampering              | [slug] URL param used in getByUID   | accept      | getByUID only queries Prismic's own API with the slug as UID; Prismic returns 404 for unknown UIDs — no arbitrary URL fetching, no SSRF   |
| T-02-10   | Information Disclosure | Preview cookie (io.prismic.preview) | mitigate    | Preview refs are cryptographically signed by Prismic; cannot be forged by attacker; preview.astro reads but does not log the cookie value |
