<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Prismic preview cookie → SSR route | Cookie value from browser is read by preview.astro; passed to resolvePreviewURL — Prismic validates the ref |
| Prismic slug param ([slug]) → getByUID | Actually, getPages() is called in getStaticPaths at build time — all valid slugs are known at build time; no runtime slug injection |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-18 | Tampering | preview cookie value passed to resolvePreviewURL | mitigate | Prismic preview refs are cryptographically signed server-side by Prismic; a forged/modified cookie results in a 403 from Prismic, not an unsafe redirect |
| T-02-19 | Information Disclosure | homepage.data.cta_url rendered as href | accept | URL comes from Prismic editorial field (link type); content authors control this; not user input |
| T-02-20 | Tampering | set:html in SliceZone with Prismic RichText | accept | Same rationale as T-02-13 — CMS editorial content, not end-user input; acceptable for PoC |
| T-02-21 | Elevation of Privilege | export const prerender = false on preview.astro | accept | SSR on this single route requires the Cloudflare adapter; all other pages remain static. Adapter is installed in plan 02-01. |
</threat_model>
