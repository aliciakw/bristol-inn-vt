## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Hostaway API → build runtime | API responses are parsed and typed; no raw data reaches client HTML |
| HOSTAWAY_ACCESS_TOKEN → fetch header | Token flows from astro:env/server into Authorization header; never logged |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-05 | Information Disclosure | HOSTAWAY_ACCESS_TOKEN in logs | mitigate | normalizeRoom logs only image URLs, never the token; getRooms/getRoom do not log headers |
| T-02-06 | Tampering | Raw API response shape | mitigate | Response cast via TypeScript interface (RawListing); normalizeRoom produces clean typed output — malformed fields result in build error, not silent corruption |
| T-02-07 | Denial of Service | Hostaway API rate limit (15 req/10s per IP) | accept | Build fetches once per room at build time for a small inn (<10 rooms); well within limits |
</threat_model>
