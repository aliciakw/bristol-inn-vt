````markdown
# Deployment Workflow — Bristol Inn

## Overview

Bristol Inn is deployed to Cloudflare Pages with automatic builds on:

- **Production:** Push to `main` branch → auto-builds and deploys to bristol-inn.pages.dev
- **PR Previews:** Push to feature branch + open PR → auto-builds and deploys to preview URL

## Prerequisites

- GitHub repository linked to Cloudflare Pages
- Cloudflare account with Pages project created
- Environment variables configured in Cloudflare project settings

## Cloudflare Pages Configuration

**Project name:** bristol-inn  
**Production branch:** main  
**Build command:** `npm run build`  
**Build output directory:** `dist`  
**Production URL:** https://bristol-inn.pages.dev

### Environment Variables (Configured in Cloudflare UI)

| Variable          | Type   | Phase Added | Value                          |
| ----------------- | ------ | ----------- | ------------------------------ |
| HOSTAWAY_API_KEY  | Server | Phase 2     | Real API key from Hostaway     |
| PRISMIC_TOKEN     | Server | Phase 2     | Real access token from Prismic |
| PUBLIC_SENTRY_DSN | Client | Phase 6     | Real DSN from Sentry           |
| PUBLIC_GA4_ID     | Client | Phase 6     | Real measurement ID from GA4   |

Server-only variables (no PUBLIC* prefix) are not exposed to client code. Client-visible variables (PUBLIC* prefix) are safe to expose.

## Deployment Workflow

### Local Development

```bash
# Start dev server
npm run dev

# Verify locally before pushing
npm run build
npm run preview

# When ready to deploy
git add .
git commit -m "feature: description"
git push origin main
```
````

### Automatic Production Deployment

1. Push to `main` branch
2. GitHub webhook triggers Cloudflare Pages build
3. Cloudflare runs `npm run build`
4. Build succeeds → Output deployed to bristol-inn.pages.dev (2-3 min)
5. If build fails → Check Cloudflare build log for errors

### PR Preview Deployment

1. Create feature branch locally: `git checkout -b feature/my-feature`
2. Make changes and push: `git push origin feature/my-feature`
3. Cloudflare Pages automatically builds preview
4. Create Pull Request on GitHub
5. GitHub shows Cloudflare Pages preview link in PR checks
6. Review preview URL to verify changes work before merging

### Rollback (If Production Build Fails)

1. Go to Cloudflare Pages dashboard → Deployments tab
2. Find the last successful deployment
3. Click "Rollback to this deployment"
4. Cloudflare will revert to previous working version (< 1 minute)

## Pre-Deployment Checklist

Before pushing to `main`:

- [ ] Run `npm run build` locally → succeeds with no errors
- [ ] Run `npm run type-check` → no TypeScript errors
- [ ] Run `npm run dev` → dev server starts without errors
- [ ] Test all routes locally:
  - [ ] http://localhost:3000/
  - [ ] http://localhost:3000/rooms
  - [ ] http://localhost:3000/contact
- [ ] Check browser console (F12) → no red errors
- [ ] Verify responsive design:
  - [ ] 375px mobile viewport → single-column
  - [ ] 640px tablet viewport → multi-column
  - [ ] 1024px desktop viewport → full layout
- [ ] No `.env.local` changes committed (should be git-ignored)
- [ ] `.env.example` updated if env var requirements changed

## Troubleshooting

### Build Fails with "Missing critical environment variables"

**Cause:** Environment variables not configured in Cloudflare Pages project settings.

**Fix:**

1. Go to Cloudflare Pages project → Settings → Environment variables
2. Add all required variables with correct values
3. Trigger manual rebuild: Deployments tab → latest build → Retry build

### Build Fails with TypeScript Errors

**Cause:** TypeScript strict mode detected a type error.

**Fix:**

1. Pull latest code locally
2. Run `npm run build` to see same error
3. Fix the TypeScript error in your editor
4. Test with `npm run dev`
5. Commit and push fix

### Production Site Shows Placeholder Content

**Cause:** Latest code deployed but Cloudflare edge cache still serving old version.

**Fix:**

1. Clear Cloudflare cache:
   - Go to Cloudflare dashboard → your domain
   - Caching tab → Purge everything
   - Wait 5 minutes for cache to clear
2. Hard refresh browser: Cmd+Shift+R (Mac) or Ctrl+Shift+R (Windows)
3. If still cached, wait up to 24 hours for TTL to expire

### PR Preview URL Not Working

**Cause:** Cloudflare build failed for feature branch.

**Fix:**

1. Check Cloudflare Deployments tab
2. Find the deployment for your branch
3. Click to see build log
4. Look for error message (usually env var or type error)
5. Fix the error locally and push again
6. Cloudflare will auto-rebuild

## Monitoring

### Build Health

- Check Cloudflare Pages dashboard daily
- Monitor build times (should be < 90 seconds)
- Alert on failed builds (configure in Cloudflare notifications)

### Production Monitoring

- Monitor Sentry for JavaScript errors (Phase 6)
- Monitor GA4 for page view tracking (Phase 6)
- Monitor Cloudflare for DDoS/security alerts

````
## Trust Boundaries

| Boundary | Description | Risk |
|----------|-------------|------|
| Local Dev → GitHub → Cloudflare | Code pushed via GitHub to Cloudflare | Mitigate: HTTPS for all transfers; GitHub OAuth; Cloudflare has verified Astro |
| Cloudflare Build → Environment Variables | Secrets loaded at build time on Cloudflare | Mitigate: Vars configured in Cloudflare UI (not in code); logs don't expose values |
| Production Static HTML → Browser | Served via Cloudflare CDN | Mitigate: HTTPS enforced; static content only (no RCE risk) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Disclosure | Env vars exposed in Cloudflare build logs | mitigate | Cloudflare automatically masks env var values in logs; use PUBLIC_ prefix for client vars only |
| T-02-02 | Tampering | Unauthorized pushes to main branch | mitigate | GitHub branch protection rules (set by user); require PR reviews before merge |
| T-02-03 | Repudiation | No audit trail of deployments | accept | Rationale: Cloudflare deployment logs + GitHub commits provide audit trail; sufficient for PoC |
| T-02-04 | Denial | Cloudflare Pages outage | accept | Rationale: Cloudflare's uptime SLA 99.95%; fallback to manual deployment if needed |
| T-02-05 | Disclosure | GitHub OAuth token compromised | accept | Rationale: User manages GitHub credentials; Cloudflare GitHub App integration is standard and verified |

**Phase 2 adds threat T-03: Hostaway API keys exposed in Cloudflare logs.** Mitigation: Cloudflare masks values; never log raw API responses.

</threat_model>
````
